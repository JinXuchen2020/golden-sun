## Phase 0 V5: 双端支持 (StorageBackend)

为桌面端 + wasm32 网页端统一存档接口：

### StorageBackend trait (src/engine/storage.rs)
```rust
pub trait StorageBackend {
    fn save(&mut self, key: &str, data: &[u8]) -> GameResult<()>;
    fn load(&self, key: &str) -> GameResult<Option<Vec<u8>>>;
    fn exists(&self, key: &str) -> bool;
    fn delete(&mut self, key: &str) -> GameResult<()>;
}
```

### 双端实现
- `FsStorage` (desktop): 读写本地文件，含文件名安全过滤 + 路径消毒
- `LocalStorage` (wasm32): 浏览器 localStorage，base64 编码绕过 UTF-16 限制
- `create_storage()` 工厂函数：根据编译目标自动选择后端

### Cargo.toml 配置
- wasm32 依赖: `web-sys` (Window + Storage features)，条件编译
- crate-type: `["rlib"]` (wasm 构建用 `--lib`)

### 集成
- `main.rs` GameCtx 中 `storage: Box<dyn StorageBackend>` 
- Phase 6 存档/读档直接 `ctx.storage.save("save", &data)?`
